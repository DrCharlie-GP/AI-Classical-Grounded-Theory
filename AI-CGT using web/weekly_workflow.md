# 一周工作节奏

## 节奏的基本原则

经典扎根理论需要浸润，浸润需要慢。当代 LLM 工具的便利性最大的风险不是它说错什么，而是**它让研究者总是可以"问一下"**——这让研究者独自与资料相处的时间被不断打断，浸润从未真正发生。

本节奏的核心设计是：**在一周的 5 个工作日中，有 4 天研究者不与 LLM 做研究相关对话**。LLM 的介入集中在一个时段——周五下午——作为一周工作的整理与检验。这不是技术限制，是方法论纪律。

次要原则：**每周五留出 2 小时**专门更新 `research_state.md` 并上传到 Claude Project Knowledge。这 2 小时是研究者对一周工作的反思，对研究的推进极为关键，**不应以"本周没什么可更新"为由跳过**。

---

## 周一：资料收集日

**上午**：访谈 1-2 位受访者（若研究处于资料收集阶段）。访谈前不看任何之前的备忘录——以"干净"的状态进入访谈，避免让已形成的范畴过早指引提问方向。

**下午**：回到办公室，整理访谈的关键词笔记为完整的访谈记录。若采用录音+转录路径，运行转录流程。**不做编码，不写备忘录**——只是把资料整理成可编码的形式。Glaser（1978）建议在访谈结束当天尽早完成记录整理，以免细节流失。

**晚上（可选）**：写一份简短的田野日志——不是备忘录，是研究者当天的主观感受、对受访者的印象、任何情绪反应。这份日志属于"一切皆为数据"中研究者本身的体验，日后回看时可能浮现出意想不到的关联。

**LLM 使用**：无。

---

## 周二至周三：独立浸润日

**全天**：对周一收集的资料做开放编码。逐行/逐句阅读，只用 REFERENCE §2 的五个中性问题。每浮现一个想法立即写备忘录。每次标记编码时手写在本地（或按研究者自己的技术工具登记）。

中午可以做一次"换视角"——把昨天写的备忘录重读一遍，看看在今天的阅读之后是否有新联想。这是模拟 Glaser 所说的"不同时间重读同一份资料"的微型版本，虽然时间跨度小得多。

**LLM 使用**：原则上无。例外情况——若遇到具体的方法论疑问（不是对资料的判断问题），可以做一次简短的类型 A 对话（不超过 15 分钟）。例如"我在犹豫是否应该把'向下比较'与'社会比较'合并为一个范畴，SKILL 对这类合并判断有什么指导"——这是方法论咨询，不涉及对资料的判断，可用。但要警惕这类对话的频率，若一周超过 2 次类型 A 对话，说明研究者对方法论的掌握可能需要回归 SKILL 与 REFERENCE 重读，而非依赖对话。

---

## 周四：理论性采样决定日

**上午**：继续周二至周三的开放编码工作。

**下午**：做本周的理论性采样决定。根据本周浮现的概念，决定下周找什么样的受访者。这一决定应书面化记录在 `logs/sampling_decisions.md` 中：本次采样的驱动概念、预计接触对象、选择理由、预期比较维度。

如需与导师或课题组讨论采样决定，在周四下午完成讨论——**讨论的对象是人，不是 LLM**。课题组的人工讨论在经典扎根理论中扮演不可替代的角色。

**晚上**：预备下周的访谈联系与安排。

**LLM 使用**：无。

---

## 周五：整理与反思日

这是一周工作最特殊的一天。它分为三段。

### 周五上午：本周备忘录整理

整理本周新写的所有备忘录。对每份备忘录：

- 确认标题是它所围绕的范畴名
- 确认内文标记了涉及的其他范畴
- 决定它的归档位置（哪个范畴子目录）
- 对质量明显不足（例如只是描述、未达概念化）的备忘录，修订或标记为"草稿"

整理过程中不与 LLM 对话。这是研究者独自重读本周工作的时刻，浸润的核心环节。

### 周五中午：更新 research_state.md

基于本周的工作更新 `research_state.md`。重点更新的节：

- §1 资料累计情况
- §3 已浮现的范畴（新增/修改）
- §4 中心范畴候选（若已进入选择性编码阶段）
- §5 当前的理论性采样指向
- §6 当前的关键方法论疑问
- §7 备忘录清单
- §8 本周的变化
- §10 AI 协作状态

更新完成后将新版本覆盖上传到 Claude Project Knowledge，旧版本以 `research_state_YYYY-MM-DD.md` 格式在本地归档。

### 周五下午：LLM 对话时段

本周与 LLM 的集中对话时段，一般 1-2 次，每次 30-60 分钟。

**每周都做的**：根据本周实际情况选择 1-2 种对话类型。常见组合：

- 若本周有浮现需要验证的概念：类型 B（不断比较素材对话）。
- 若本周有备忘录草稿需要语言质询：类型 C（备忘录写作陪练），但一周不超过 2 次。
- 若本周有具体的方法论疑问积累：类型 A（方法论咨询对话），合并问题在一次对话中问完。

**每 2-3 周做一次**：类型 D（阶段性反思对话）。这类对话占用较多时间（约 1-1.5 小时），建议排在本周没有其他对话时单独做。

**对话后**：按 `dialogue_log_TEMPLATE.md` 归档每一次对话。若对话激发了新备忘录想法，立即独立手写备忘录——不复制 LLM 的产出。

### 周五晚上：封存一周

把本周的所有文件（`data/`、`memos/`、`logs/`、`dialogue_logs/`）做一次 Git 提交（若使用版本控制）或完整备份。在 `methodological_memos.md` 中写一段 50-150 字的本周总结。

---

## 周六：休息

休息。

不例外。CGT 研究是长跑，研究者的身心状态与浸润能力直接相关。Glaser（1978）未明确规定节奏，但他反复强调"让概念有时间浮现"——浮现需要的不仅是阅读资料的时间，也包括研究者大脑在非工作状态下的消化时间。洗澡、散步、看不相关的书、打球、陪家人——这些时段可能正是某个未来的洞察在无意识中成形的时段。

**LLM 使用**：无。

---

## 周日：轻度准备日

周日下午可做 1-2 小时的轻度准备：回顾下周的访谈安排、检查录音设备（若使用）、翻阅本周的备忘录（不做新写作）。

这一天的目的是让研究者以"就绪但未紧绷"的状态进入下周一。**不做任何需要深度判断的工作**。

**LLM 使用**：无。

---

## 节奏的例外处理

### 进入选择性编码阶段后

当中心范畴已明确浮现、研究进入选择性编码阶段后，节奏微调：

- 周二至周三的开放编码改为选择性编码（只对与中心变量相关的概念编码）。
- 周五下午可能增加一次类型 B 对话（检验中心范畴在更多资料中的 fit）。
- 每 2 周一次的类型 D 对话在此阶段特别重要，用于检验中心范畴是否真正满足十一项判据。

### 进入外部比较阶段后

- 周二至周三可开始文献阅读（这是被延迟的文献回顾正式开始的时点）。
- 周五下午的 LLM 对话可能加入新类型：让 LLM 协助检索与中心范畴相关的文献（这是一种类型 A 的变体，让 LLM 做方法论+检索辅助）。注意——LLM 只做"找到文献"，文献与研究概念的比较判断由研究者自己做。
- 文献停车场中的条目在此阶段按标签召回，由研究者逐一审视。

### 进入写作阶段后

- 周一至周四的访谈与编码停止（除非因 modifiability 需要新采样）。
- 大部分时间用于整理（手工 sorting）备忘录，形成章节大纲。
- LLM 对话类型增加"写作陪练"（类型 C 的扩展版，针对论文段落而非备忘录）。

## 对这个节奏的自检问题

每月底，问自己以下几个问题：

**1. 本月我独自与资料相处的总时长是多少？与 LLM 对话的总时长是多少？前者应至少是后者的 5 倍。**

**2. 本月的每次 LLM 对话结束后，我都能说出"我从这次对话中形成了什么新判断"吗？如果多次对话答不上来，说明对话正在成为消遣而非工作。**

**3. 本月我是否有至少一次"在散步/洗澡/睡前"想到研究的时刻？**这是浸润的信号。如果完全没有，说明研究者与资料的接触停留在工作时段的表层，浸润尚未发生。

**4. 我本月的备忘录写作量是否稳定？**Glaser 建议备忘录是"captures the frontier of the analyst's thinking"——每次被 spark 就写。一周写 3-5 份备忘录是正常的研究节奏。一周 0-1 份可能说明研究者处于僵局或浸润不足。

**5. 我的 `research_state.md` 中"本周的变化"一节是否真的每周都有实质变化？**如果连续 2 周填不出变化，说明研究停滞，需要考虑是什么阻碍了浸润——是资料不够丰富？是研究者对方法论不熟悉？是生活状态问题？

## 节奏的哲学

这个节奏看起来慢、低频、甚至有些"浪费"了 LLM 的能力。但它对应 CGT 的一个核心信念：**理论不是被挖出来的，是被等出来的**。Glaser 反复使用 emergence 这个词——浮现——它暗示理论不是由研究者的努力直接产生的，而是在研究者保持正确的状态时自己到来的。

正确的状态包括耐心、包括容忍不确定、包括对资料的真诚好奇、包括对自己初步想法的怀疑。这些状态都需要**时间**和**不被打扰的连续性**来培养。LLM 对话的便利性，若不加限制，会让每一次可能的浮现都被对话的"快速反馈"切断。

所以这个节奏的本质是：**用技术上的自我限制来保护方法论的核心承诺**。当您感到"我今天真的不需要 LLM 帮我做任何事"的时候，那是一个好的信号。

---

*文件结束。本文件为推荐节奏，研究者可根据自己的研究阶段与生活状态调整，但"LLM 使用时间远少于独自工作时间"这一原则不应妥协。*
